# n8n Workflow Factory

> 一个 n8n 工作流开发、管理、部署平台

## Session Start Here

新开 session 时，先按这个顺序读：

1. `AGENTS.md`
2. `KNOWLEDGE.md`
3. 对应产品目录下的 `README.md`

当前主产品是：
- `workflows/po-parser`

如果任务涉及线上 n8n / workflow 调试 / 生产修复，只读产品 README 不够，必须同时看：
- `workflows/po-parser/workflow.json`
- `workflows/po-parser/service/po_parser_service.py`

`AGENTS.md` 记录了新 session 上手顺序、服务器连接方式、生产 workflow ID、常见坑和调试捷径；`KNOWLEDGE.md` 记录了 n8n API、表达式、节点兼容性和排障经验。

## 项目定位

本项目是一个 **n8n 工作流工厂**，用于：

1. **生产工作流产品** - 每个业务场景对应一个独立的工作流产品
2. **管理组件资产** - 自定义节点、模板、配置的可复用管理
3. **标准化开发流程** - 从需求到部署的完整流水线

## 快速开始

### 环境要求

- Node.js >= 18
- n8n >= 1.0
- Python >= 3.9 (可选，用于部分解析功能)

### 安装依赖

```bash
npm install
```

### 开发新工作流

1. 在 `workflows/` 下创建新产品目录
2. 参考 `docs/development.md` 进行开发
3. 使用 `scripts/deploy.sh` 部署到 n8n
