## 核心设计原则

### 1. 产品化

每个工作流是一个独立"产品"，具备：
- 明确的业务价值
- 完整的文档
- 可测试的输入输出
- 版本管理

### 2. 组件化

可复用的组件包括：
- **自定义节点**：封装复杂逻辑
- **模板**：标准化工作流模式
- **Schema**：数据契约定义

### 3. 标准化

统一的：
- 目录结构
- 命名规范
- 配置格式
- 文档模板

## 产品生命周期

```
需求分析 → 设计 → 开发 → 测试 → 部署 → 监控 → 迭代
   │                                                    │
   └────────────────────────────────────────────────────┘
                        持续改进循环
```

## 技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 工作流引擎 | n8n | 开源、可视化、可扩展 |
| LLM | Ollama + Qwen | 本地部署、中文支持好 |
| PDF 解析 | pdfplumber + LLM | 混合方案 |
| OCR | Ollama 视觉模型 | 本地处理 |
| SAP 集成 | RFC / 中间服务 | 待确认 |
| 文件存储 | SMB 共享 | 公司现有基础设施 |

## 扩展性设计

### 水平扩展

- 新增产品：复制目录结构，修改配置
- 新增节点：开发自定义节点，放入 nodes/

### 垂直扩展

- n8n 队列模式：高负载场景
- 数据库升级：SQLite → PostgreSQL

## 安全设计

1. **凭证管理**：使用 n8n 内置凭证系统
2. **环境隔离**：敏感配置使用环境变量
3. **访问控制**：n8n 用户权限管理
4. **审计日志**：记录所有操作
